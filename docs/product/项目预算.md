# 绿境智慧AI园丁 — 各阶段项目预算

> 基于 [项目计划书 - 详细](../../raw/绿境智慧AI园丁/项目计划书%20-%20详细.md) 中的 7.2 产品发布节奏与 §8 财务预测推导。
> 团队采用复合型工程师 + 高度自动化的研发模式，规模刻意精简。
> 金额单位：**人民币万元**，一线城市口径，含五险一金（1.4× 系数）。

## 四、成本结构（24 个月累计 ≈ 2,560 万）

| 大类 | 金额（万） | 占比 |
|------|-----------|------|
| 人员（含五险一金） | ~1,160 | 45% |
| 硬件 BOM + 履约 | ~910 | 36% |
| 营销获客 | ~475 | 19% |
| 模型调用 + 云服务 + 工具 | ~285 | 11% |
| 内容/知识库/专家 | ~50 | 2% |
| 法务/办公/杂项 | ~80 | 3% |

> 各项加总略超 100%，因部分小项（如研发期模型迭代）在结构表中归入"模型/云/工具"大类，明细中归入软件研发口径。

**结构特征**：
- **软件阶段（P1–P3）**：人员占比 70%+，是典型的轻资产研发模型。
- **硬件阶段（P4–P5）**：BOM + 履约迅速攀升至 30%+，需特别注意现金流与库存。
- **生态阶段（P6）**：营销占比上升到 20%+，需配套精细化运营降本。

## 七、关键判断

1. **天使轮 200–300 万即可支撑 12 个月**，跑到 50 万用户 + 3% 付费率，创始人稀释最小化。
2. **A 轮 1,500 万左右可覆盖硬件量产+市场扩张**，不必背 3,000 万级别的融资压力。
3. **核心瓶颈是人**：1–2 个复合型资深工程师 + 高质量知识库 + 硬件供应链能力，比资金更稀缺。
4. **模型调用开销是必投项**，缩减会导致研发节奏和产品体验同时下滑。
5. **建议同时维护一份保守版**（团队 +30%、调用 +50%）作为下限模型，与本预算形成上下限区间。
