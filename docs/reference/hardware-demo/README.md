## 盆上环境监测仪（中文界面 · 高档造型 · 传感器触脚）

与「绑带 / 胶贴」方案不同，本方向强调 **传感器触脚 = 探针 + 支撑**：多根金属探针自主机底部伸出，**插入盆土** 后利用土壤阻力与主机重心形成 **自稳三角（或四角）结构」，机身悬于盆口一侧，整体观感接近 **高端园艺仪器 / 桌面摆件**。  

- **界面语言**：量产固件与 Demo 视频统一 **简体中文**（温度、湿度、光照、土壤 pH 等）；图标（蓝牙、电量）可中英通用。  
- **造型档次（CMF 方向）**：拉丝或微喷砂金属中框、窄金色倒角、曲面玻璃盖板、哑光深绿背壳；避免廉价大塑料感。  
- **触脚结构（工程提醒）**  
  - 触脚内可集成：**土壤湿度 / 温度 / pH 电极通道**（pH 与参比脚位需与结构共设计，避免串扰）。  
  - 需做 **插入深度止位** 与 **根部弯折释放圆角**，防止用户插过头折断或裂盆。  
  - 盆径过小或基质过松时自稳变差——路演可配 **官方展示盆 + 标准基质** 保证观感。  
- **屏幕**：高档路线常见 **OLED / 高对比 TFT**；若走长续航可另做 **墨水屏 SKU**。  
- **传感与话术**：pH 仍为 **趋势级** 精度；光照注意叶面遮挡；与软件云端可对齐为「本地读数 + 云端建议」双源。  
- **通信**：Demo 期 **BLE → 小程序**；量产再评估网关。

## 设计意图（实现时可裁剪）

1. **智能花盆**  
   - 储水层 + 水位视窗，降低「忘记浇水」的演示门槛。  
   - 探针仅做 **湿度 + 温度** 即可对齐现有软件里的 `SoilMoistureHint` / 环境叙事；肥力若不做硬件传感器，可继续用 **拍照 LLM** 补充。  
   - 边缘模块：BLE 优先（与手机直连 Demo），量产再考虑 Wi-Fi。

2. **传感棒**  
   - 不做盆体也能单独 Demo：**插土 → 读数 → 手机/小程序展示**（可接现有后端扩展一条「设备上报」API，属后续 change）。  
   - OLED 条屏只做 1～2 个核心字段，避免 UI 过重。

3. **家庭网关（可选）**  
   - 路演用「多设备汇聚」叙事；早期可用 **手机 BLE 直连** 替代网关以省 BOM。

## 与软件栈的对齐说明（口头即可）

- 天气、订阅提醒、计划引擎仍以 **当前后端 + 小程序** 为准。  
- 硬件侧只增加 **新的数据来源通道**（设备上报湿度/温度），不改变现有 MVP 契约。
